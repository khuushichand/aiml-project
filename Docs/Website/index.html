Core capabilities

Built for the full research loop

tldw is not just a transcript viewer or a thin chat wrapper. It is an open-source research assistant that keeps ingestion, retrieval, and analysis on the same footing.

Ingest almost anything

Video, audio, PDFs, EPUB, DOCX, HTML, Markdown, XML, and web content from thousands of sources.

Transcribe and synthesize speech

Real-time and file-based speech workflows across multiple STT and TTS providers with local options included.

Search with Unified RAG

Hybrid BM25 and vector retrieval, reranking, and a retrieval model built for actual research workloads.

Use local or hosted models

Run local stacks or bring commercial providers behind one OpenAI-compatible interface.

Keep artifacts, prompts, and notes together

Chatbooks, prompt tooling, notes, flashcards, and research outputs all stay connected instead of getting scattered.

Operate on infrastructure you control

Self-hosted defaults, privacy-first posture, and no telemetry baked into the public story.

Why self-host

Privacy should not depend on someone else's SaaS roadmap.

tldw is designed for people who want a serious research tool without handing source material, voice data, or internal knowledge workflows to an opaque third party.

Keep your data local

Store transcripts, notes, and artifacts on infrastructure you administer instead of whatever a default cloud keeps around.

No telemetry by default

The product story stays consistent with the project story: your work is not a data-harvesting side effect.

Audit what you run

Open-source code, explicit configuration, and documented deployment profiles make the system inspectable.

Quickstart

Get a working stack up fast

The default path should stay practical: the Docker single-user + WebUI setup most people actually want.

Recommended: Docker + WebUI 
git clone https://github.com/rmusser01/tldw_server.git
cd tldw_server
make quickstart

Launches the API at http://localhost:8000 and WebUI at http://localhost:8080.

Need the developer-oriented local install path instead? Use make quickstart-install after cloning.

Running a shared deployment? Start with Profile_Docker_Multi_User_Postgres.md.

Alternative: API only 
git clone https://github.com/rmusser01/tldw_server.git
cd tldw_server
make quickstart-docker

Use this when you want the production-style API path without the WebUI layer.
