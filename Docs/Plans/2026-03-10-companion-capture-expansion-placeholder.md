# Companion Capture Expansion Placeholder

## Purpose

Reserve the next companion breadth-oriented slice so it does not get lost after the quality and trust-controls milestone.

## Why Deferred

The companion foundation already has enough explicit event sources to be useful. The higher-value gap is currently relevance, provenance, and user trust controls.

Expanding capture breadth before quality improves would make the system noisier without making it more dependable.

## Intended Scope

Potential future work:

- more extension-side explicit actions
- more adjacent-system capture bridges
- more structured companion capture affordances across reading, collections, watchlists, and other user-initiated flows

## Preconditions

This placeholder should not move forward until the quality/trust-controls milestone is complete:

- query-aware bounded context selection
- richer derivations and reflections
- provenance drill-down
- scoped purge/rebuild controls

## Deliverable Expectation

When activated, this should become a full design and implementation plan pair, not an ad hoc feature drip.
