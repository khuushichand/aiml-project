## 1. Search Experience (SearchBar)

| # | Finding | Severity | Type | Current State | Recommendation |
|---|---------|----------|------|---------------|----------------|
| 1.1 | Search bar is prominent and well-centered on first load. The BookOpen icon + "Knowledge QA" heading + descriptive subtitle creates a good first impression. The centered-to-top transition on results is smooth. | -- | -- | `flex-1` centers vertically when no results; `pt-6 pb-4` on results. Transition uses `transition-all duration-300`. | No change needed. Good implementation. |
| 1.2 | Rotating placeholder examples cycle every 8 seconds and stop after 3 full rotations. The 4 examples are all document-analysis queries, which may not demonstrate the full range of capabilities (e.g., no cross-document comparison, timeline, or factual lookup examples). | Nice-to-Have | Information Gap | `EXAMPLE_QUERIES` has 4 items; cycles at 8s intervals; stops at `MAX_CYCLES = 3`. Pauses on focus. | Add 2-3 more diverse examples: "Compare the conclusions across my PDFs", "When was X first mentioned in my sources?", "What are the citations for claim Y?" |
| 1.3 | Keyboard shortcut hints are shown below the search bar as `kbd` elements. The `/` to focus and `Cmd+K` for new search are well-implemented. However, the hints are in `text-xs text-text-muted` which may be too subtle for first-time users. | Nice-to-Have | UX/Usability Issue | `kbd` elements with `px-1.5 py-0.5 bg-muted rounded font-mono`. Shown in a `flex justify-between` row below input. | Consider showing hints more prominently on first visit (e.g., slightly larger text or a brief animation), then reducing to current subtlety. |
| 1.4 | The "Ask" submit button uses `disabled:opacity-50` for the empty-query state. The disabled state is clear. However, during search the button text changes to `"..."` which is cryptic. | Important | UX/Usability Issue | `isSearching ? "..." : "Ask"`. Button is disabled during search. | Change to a more informative state: show a small spinner icon or "Searching..." text to match the visual language of the rest of the UI. |
| 1.5 | Web Search Fallback toggle is a small text button below the search bar with `aria-pressed` state. The visual distinction between on/off is good (primary color vs border-only). However, there is no tooltip or description explaining *when* web fallback activates. | Important | Information Gap | Toggle reads "Web search on/off" with `title="Toggle web search fallback"`. No description of threshold behavior. | Add a tooltip: "Falls back to web search when local source relevance is below threshold (configurable in settings)". |
| 1.6 | Loading state replaces the Search icon with a spinning Sparkles icon and disables the input with `opacity-75`. The Sparkles animation is subtle and may not be noticed by users focused on the input text. | Important | UX/Usability Issue | `animate-spin` on `Sparkles` icon. Input gets `opacity-75 cursor-not-allowed`. | Add a more prominent loading indicator: consider a progress bar below the input or a pulsing border effect. | 1.7 | No cancel/abort mechanism for in-progress searches. Once submitted, users must wait for the search to complete or fail. | 1.8 | Clear button (X icon) both clears the query text and clears all results via `handleClear` which calls `clearResults()`. | 1.9 | No character limit on query input. | 1.10 | Search bar has `aria-label="Search your knowledge base"` which is good. ## 2. AI Answer Display (AnswerPanel)

| # | Finding | Severity | Type | Current State | Recommendation |
|---|---------|----------|------|---------------|----------------|
| 2.1 | The answer panel has good visual hierarchy: gradient background (`from-primary/5 to-primary/10`), a header bar with Sparkles icon and "AI Answer" label, and a citation count. | 2.2 | Inline citation markers [N] are rendered as clickable buttons with distinct visual treatment: cited markers get `bg-primary text-white`, uncited get `bg-surface2 text-text-muted`. The distinction is clear. However, the markers are only 20px tall (`h-5`) which may be hard to tap on mobile. | Important | UX/Usability Issue | Citation buttons: `min-w-[1.5rem] h-5 px-1.5 mx-0.5 text-xs`. | 2.3 | Citation click scrolls to the source card via `scrollIntoView({ behavior: "smooth", block: "center" })` and applies a visual focus ring (`ring-2 ring-primary/20`). | 2.5 | Loading state shows "Searching your knowledge base..." with a subtitle "Finding relevant documents and generating an answer". | 2.6 | Error state shows the error message from the catch block but does not distinguish between error types. | 2.7 | When results exist but no generated answer (generation disabled), the hint reads "Enable answer generation in settings to get a synthesized response." | 2.8 | No copy button for the full answer. Users must manually select and copy text. | Important | Missing Functionality | No copy action in the answer panel header or footer. | Add a "Copy answer" button in the header next to the citation count. Include citations in the copied text. |
| 2.9 | No feedback mechanism (thumbs up/down) on answers. | 2.11 | No indication of whether web fallback was used vs. purely local sources. ## 3. Source Cards & Retrieved Documents (SourceList, SourceCard)

| # | Finding | Severity | Type | Current State | Recommendation |
|---|---------|----------|------|---------------|----------------|
| 3.1 | Two-column grid layout (`md:grid-cols-2`) is effective for scanning. Single column on mobile. Gap spacing at `gap-4` provides clear separation between cards. | -- | -- | `grid gap-4 md:grid-cols-2`. Responsive breakpoint at `md`. | No change needed. Good layout. |
| 3.2 | Sort toggle offers only "By Relevance" and "By Title". Researchers frequently need to sort by date, source type, or cited-first. | Important | Missing Functionality | Binary toggle between `"relevance"` and `"title"` sort modes. Relevance is default API order. | Add "By Date" and "Cited First" sort options. Consider a dropdown instead of a toggle for 4+ options. |
| 3.3 | Relevance score shows "N% match" as a small badge. The percentage is `Math.round(score * 100)`. For users unfamiliar with RAG scoring, 73% vs 45% may not convey meaningful quality difference. | Nice-to-Have | Information Gap | `score` from API rendered as percentage in `bg-muted rounded` badge. No calibration guidance. | Add a color gradient (green/yellow/red) to the relevance badge. Consider labels like "High match" / "Moderate" / "Low" alongside the percentage. |
| 3.4 | Cited vs. uncited visual distinction uses a "Cited" badge with Quote icon on cited cards plus a primary-colored index badge (vs. muted for uncited). The distinction is clear but uncited sources have equal visual weight - they are not de-emphasized. | Nice-to-Have | UX/Usability Issue | Cited cards: `bg-primary text-white` index badge + `text-primary` "Cited" label. | 3.5 | Source metadata shows title, type icon, page number, relevance score, and excerpt. Missing: date/age of document, word count of chunk, position within document (e.g., "Chunk 3 of 12"), and full document title when truncated. | Important | Information Gap | `RagResult.metadata` has `title`, `source`, `url`, `page_number`, `chunk_id`, `source_type`. | 3.6 | Excerpt truncation at 300 characters with "..." suffix. No expand/collapse mechanism to see the full chunk content. | Important | UX/Usability Issue | `truncateText(content, 300)` renders in card body. | 3.7 | "Ask About This" pre-populates `"Tell me more about {title}"` without auto-submitting. Good pattern, but the template is generic. | 3.8 | "Copy" action copies the full raw content (not the truncated excerpt). This matches researcher expectations. | 3.9 | "Open" action (external URL) is only rendered when `url` exists. When no URL exists, the button is completely absent with no placeholder or explanation. | 3.10 | Keyboard navigation with 1-9 keys, Tab cycling, and Escape to clear is well-implemented. The hint text is shown below the header. | 3.11 | No source type filtering. All results are shown in a flat list regardless of type (media_db, notes, characters, chats). | 3.12 | No pagination or virtualization. All results render in the grid simultaneously. | 3.13 | Source cards lack semantic ARIA roles. They are plain `div` elements with no `role` attribute. ## Executive Summary

### Top 5 Critical Gaps That Would Block Researcher Adoption

1. **No streaming answers (11.1)**: Researchers waiting 20-30 seconds with a static spinner and no partial output will abandon the tool. Streaming the answer incrementally is table-stakes for any LLM-powered Q&A interface.

2. **No multi-turn conversation view (11.2 / 4.4)**: The system tracks conversation messages but only displays the latest exchange. Researchers conducting deep inquiry need to see their full thread of questions and answers to build understanding progressively.

3. **No search cancellation (1.7)**: Without an abort mechanism, users are trapped during long Thorough-preset searches. This is a fundamental usability requirement for any search interface.

4. **Answer rendered as plain text, not markdown (2.4)**: AI-generated answers frequently contain lists, code blocks, bold text, and tables. Rendering everything as flat text with `whitespace-pre-wrap` severely undermines answer readability.

5. **Accessibility gaps in ExportDialog and accordion sections (12.4, 12.7, 12.8)**: Missing `role="dialog"`, missing `aria-expanded`, and hover-only delete buttons create barriers for keyboard and screen reader users. These are straightforward fixes with high impact.

### Top 5 Quick Wins (High Impact, Low Effort)

1. **Add `aria-expanded` to accordion section buttons (12.7)**: One attribute addition per button. ~15 minutes of work. Unblocks screen reader users.

2. **Add `role="dialog"` and `aria-modal` to ExportDialog (12.4)**: Two attribute additions. ~10 minutes. Critical a11y fix.

3. **Add `focus-visible:opacity-100` to history delete button (12.8)**: One CSS rule. ~5 minutes. Makes the button visible to keyboard users.

4. **Show "No results found" empty state (10.9)**: Small conditional render. ~30 minutes. Eliminates confusing blank state after a query that returns nothing.

5. **Add `aria-label` to follow-up input (12.11)**: One attribute. ~2 minutes. Screen reader accessibility fix.
